Your laptop shouldn't be the bottleneck

This is a real product. It's a rubber hand you wedge into your laptop so the lid can't close, because closing the lid would put the machine to sleep, and sleeping the machine would kill the agent that's four hours into a refactor.

Agents run for hours now, and this is where we've landed as an industry: a little orange guy, holding the lid, guarding your session. Software engineering in 2026 is checking Prevent sleep when display is off and carrying a warm backpack.

Here's the less silly fix. Harness can now connect to remote backends — headless Harness servers running on any other machine: a desktop in the other room, a Mac mini in a closet, a Linux box in the cloud. Each backend has its own worktrees, terminals, browser tabs, and settings, and they all show up in the same window as chips at the bottom of the sidebar. The agents keep running whether you're looking or not. Close the lid. The little guy can retire.

Connect to a remote in one click

If you can ssh to the machine, you're already done with the hard part. Open the + on the sidebar's backend chip strip (or File → Add Backend…), switch to the SSH host tab, and pick a host straight from your ~/.ssh/config — or type user@host. Click Connect.

Harness handles everything else over that SSH connection: probes for an existing install, runs the installer if the server isn't there yet, starts harness-server detached so it survives the disconnect, and opens a local port forward. You watch each phase stream by — connecting, probing, installing, starting, tunneling, connected — and then the chip appears. You never open a terminal on the remote machine.

The server binds to 127.0.0.1 on the remote box and all traffic rides the SSH tunnel, so nothing new is exposed on the network. Quit Harness and the server keeps running; reopen it and the backend reconnects.

What's different

One-click SSH setup

Pick a host from ~/.ssh/config, click Connect. Harness installs the server, starts it, and tunnels to it — no terminal on the remote required.

Switch with a keystroke

Click a chip, or hit ⌘⇧1..9 to jump to the Nth backend. Local is always 1.

Notifications from every machine

Inactive backends keep streaming in the background. An agent waiting for approval on the box in the closet still pings you.

Works from your phone

Every server also serves a web client. Point any browser at the http:// URL, pin it to your homescreen. Same UI, no install.

Token-authed, localhost by default

Bearer-token auth on every connection. Binds to 127.0.0.1 unless you explicitly opt into LAN. Tokens are encrypted at rest in the desktop app.

Same UI everywhere

Worktrees, terminals, chat, changed files, PR status — the full Harness surface, per backend. Even browser tabs work headlessly, via playwright-core driving system Chrome.

Prefer to set it up yourself?

The SSH flow is a convenience wrapper around a server you can run entirely by hand — useful when the machine isn't in your SSH config, or you want to manage the process yourself. Three steps: 

# 1. Install on the remote box — no Node, no deps
$ curl -fsSL https://raw.githubusercontent.com/frenchie4111/harness/main/scripts/install-headless.sh | sh

# 2. Run it
$ harness-server --port 0
[web-client] open http://127.0.0.1:53412/?token=…
[server] listening on ws://127.0.0.1:53412/?token=…

# 3. In the desktop app: chip strip → + → paste the link
#    → Test & save. Done.

The installer pulls the right tarball for the platform (darwin-arm64, linux-x64, linux-arm64), verifies the sha256, and extracts to ~/.harness-server/. It's self-contained: embedded Node, bundled claude binary, web client, MCP bridge. The printed URLs have the auth token embedded, and the token survives restarts — daemonize with nohup, tmux, or screen and paste the same link once.

Try it

The full setup — including LAN exposure, choosing a Chromium for headless browser tabs, and daemonizing — is in the headless server section of the README. One macOS note: the tarballs aren't Apple-signed, so Gatekeeper may quarantine the binary. Clear it with xattr -d com.apple.quarantine ~/.harness-server/bin/harness-server.

If you don't see the chip strip yet, update from the latest release.
