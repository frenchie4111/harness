Parallel agents

Ten Claudes.
One window.

Kick off a task, switch to the next one, and another. Harness keeps every agent in its own worktree so they never step on each other — and one glance tells you which ones are working, which are waiting, and which need you.

MCP built in

Claude spawns
its own worktrees.

A built-in MCP server gives Claude the keys to Harness. Ask it to spin up a new worktree for a refactor and it just does it — no terminal detour, no context switch, no "let me set that up for you."

Reliable status dots

Each worktree's sidebar dot shows exactly what Claude is doing — working, waiting on you, or asking permission. Powered by Claude Code hooks, not flaky output parsing.

One worktree per task

Create and delete git worktrees from the sidebar. Each task gets its own folder and branch so agents never trip over each other's edits.

Click any file for a Monaco-powered editor with syntax highlighting for every language, multi-cursor, find-and-replace, and ⌘S to save. Diffs auto-collapse unchanged regions and expand on click.

After install
  1. Pick a git repository when Harness asks.
  2. Click the gear icon to paste a GitHub personal access token (optional, needed for PR status).
  3. Click Enable on the hooks banner so Claude status detection works.
Requires claude CLI installed and git on your PATH.
