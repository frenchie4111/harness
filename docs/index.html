Reliable status dots

Each worktree's sidebar dot shows exactly what Claude is doing — working, waiting on you, or asking permission. Powered by Claude Code hooks, not flaky output parsing.

One worktree per task

Create and delete git worktrees from the sidebar. Each task gets its own folder and branch so agents never trip over each other's edits.

Activity timeline

A dedicated Activity view tracks every worktree's status history. See at a glance which agents have been productive and which got stuck.

Configurable hotkeys

⌘1–⌘9 jumps between worktrees. ⌘T opens a new shell. ⌘⇧G opens the active PR. ⌘B hides the sidebar. All rebindable in Settings.

After install
  1. Pick a git repository when Harness asks.
  2. Click the gear icon to paste a GitHub personal access token (optional, needed for PR status).
  3. Click Enable on the hooks banner so Claude status detection works.
Requires claude CLI installed and git on your PATH.
