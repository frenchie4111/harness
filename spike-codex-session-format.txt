Codex session JSONL format — captured from ~/.codex/sessions/

Each line is JSON with a "type" field. Observed types:

1. session_meta — one per file, first line
   {"type":"session_meta","payload":{"id":"<uuid>","timestamp":"...","cwd":"...","originator":"codex_cli_rs","cli_version":"0.46.0","instructions":null,"source":"cli","git":{"commit_hash":"...","branch":"main","repository_url":"..."}}}

2. response_item (type=message, role=user) — user messages
   {"type":"response_item","payload":{"type":"message","role":"user","content":[{"type":"input_text","text":"..."}]}}

3. event_msg (type=user_message) — duplicate user message signal
   {"type":"event_msg","payload":{"type":"user_message","message":"...","kind":"plain"}}

4. turn_context — per-turn metadata
   {"type":"turn_context","payload":{"cwd":"...","approval_policy":"on-request","sandbox_policy":{"mode":"workspace-write",...},"model":"gpt-5-codex","summary":"auto"}}

5. event_msg (type=token_count) — per-API-call usage
   {"type":"event_msg","payload":{"type":"token_count","info":{"total_token_usage":{"input_tokens":3076,"cached_input_tokens":2048,"output_tokens":43,"reasoning_output_tokens":0,"total_tokens":3119},"last_token_usage":{...},"model_context_window":272000},"rate_limits":{...}}}

6. event_msg (type=agent_reasoning) — reasoning summaries
   {"type":"event_msg","payload":{"type":"agent_reasoning","text":"**Checking routes for new endpoint**"}}

7. response_item (type=reasoning) — reasoning block with encrypted content
   {"type":"response_item","payload":{"type":"reasoning","summary":[{"type":"summary_text","text":"..."}],"content":null,"encrypted_content":"..."}}

8. response_item (type=function_call) — tool invocation
   {"type":"response_item","payload":{"type":"function_call","name":"shell","arguments":"{\"command\":[\"bash\",\"-lc\",\"ls\"],\"workdir\":\"...\"}","call_id":"call_..."}}

9. response_item (type=function_call_output) — tool result
   {"type":"response_item","payload":{"type":"function_call_output","call_id":"call_...","output":"{\"output\":\"...\",\"metadata\":{\"exit_code\":0,\"duration_seconds\":0.2}}"}}

Key differences from Claude Code session format:
- Claude: type=user/assistant, message.content has text/thinking/tool_use/tool_result blocks, message.usage has token counts
- Codex: type=response_item/event_msg/turn_context, function_call/function_call_output are separate lines, token counts in event_msg
- Claude: one file per session at ~/.claude/projects/<encoded-cwd>/<uuid>.jsonl
- Codex: one file per session at ~/.codex/sessions/YYYY/MM/DD/<name>-<uuid>.jsonl
- Claude: usage fields: input_tokens, output_tokens, cache_read_input_tokens, cache_creation_input_tokens
- Codex: usage fields: input_tokens, cached_input_tokens, output_tokens, reasoning_output_tokens
